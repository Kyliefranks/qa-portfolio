# Test Plan - Sauce Demo Application

## 1. Introduction
This test plan defines the scope, approach, resources, and schedule for testing the Sauce Demo web application. The goal is to ensure the application functions correctly and meets quality standards.

---

## 2. Test Items
The following features will be tested:
- Login functionality
- Product listing page
- Sorting functionality
- Shopping cart
- Checkout process

---

## 3. Objectives
- Verify that users can log in with valid credentials
- Ensure error messages appear for invalid login attempts
- Confirm products can be added to cart correctly
- Validate checkout process works end-to-end
- Identify defects in UI and functionality

---

## 4. Scope

### In Scope:
- Functional testing
- UI testing
- Negative testing
- Basic usability testing

### Out of Scope:
- Performance testing
- Security penetration testing
- Automation testing (for now)

---

## 5. Test Approach
Manual testing will be performed using exploratory and structured test cases. Test cases will be executed step-by-step and results documented.

---

## 6. Test Environment
- Browser: Chrome (latest version)
- Device: Desktop/laptop
- Application: https://www.saucedemo.com

---

## 7. Entry Criteria
- Application is accessible
- Test cases are prepared
- Test environment is stable

---

## 8. Exit Criteria
- All critical test cases executed
- No open high-severity defects remaining
- Test results documented

---

## 9. Deliverables
- Test Plan document
- Test Cases
- Bug Reports
- Test Summary

---

## 10. Risks
- Application downtime
- Browser compatibility issues
- Missing requirements documentation
